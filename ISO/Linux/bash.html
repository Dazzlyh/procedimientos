USO DEL TERMINAL DE LINUX

Anatomía del comando

comando [opciones] [argumentos]

Comandos de orientación

pwd - indica el lugar donde estamos

ls - lista todo el contenido de donde estamos

  • ls -l - Muestra los archivos con información (permisos, fecha, usuario, etc)
  • ls -a - Muestra también los ficheros ocultos
  • ls -lh - Muestra lo mismo que con -l pero más legible

Comandos de navegación

cd - sirve para moverse a una carpeta

  • cd "carpeta" - nos movemos a la carpeta que indiquemos
  • cd "ruta" - nos movemos a la ruta que indiquemos
  • cd .. - volvemos un paso atrás en la ruta
  • cd . - hace referencia al mismo directorio donde estamos
  • cd ~ - hace referencia a la raíz del usuario

Rutas relativas y absolutas

ruta relativa - es relativa a la ubicación actual

ruta absoluta - es una ruta que no es relativa a la actual, te puedes ir a cualquiera escribiendo la ruta completa

Comandos básicos

whoami - te indica el nombre del usuario actual

cal - muestra el calendario del sistema

date - indica la fecha y hora actuales

uptime - indica el tiempo activo del equipo

uname - nombre del kernel (-a para más información del sistema)

clear - limpia el terminal

Ayuda y documentación

man [comando] - se abre la documentación del comando indicado (manual)

[comando] --help - muestra la información de ayuda sobre el comando

Tipos de directorios

  • / Raíz - Es el Home del SO
  • /home - directorio de carpetas personales de cada usuario
  • /etc - ficheros de configuración del sistema
  • /bin - ubicación de los programas básicos
  • /usr - ubicación del software adicional
  • /var - archivos variables (logs, colas de mensajes...)
  • /temp - archivos temporales

Manipulación de archivos

mkdir [nombre_carpeta] - Crea una carpeta en la ubicación actual

rmdir [nombre_carpeta] - Elimina una carpeta vacía

cp [archivo] [archivo_nuevo.ext] - crea una copia del archivo con el nuevo nombre en la ubicación actual

cp -r (recursivo) [carpeta] [nueva_carpeta] - Para carpetas con contenido, copia la carpeta con su contenido en una nueva

cp -a (absoluto) [carpeta] [nueva_carpeta] - Copia todo de la carpeta en la nueva, hasta archivos ocultos y aunque esté vacía

mv [nombre_archivo] [ruta_nueva] - Mueve el archivo/carpeta indicado a la nueva ruta

mv [nombre_archivo] [nuevo_nombre] - para cambiar el nombre del archivo

rm -r [carpeta] - borra el directorio y todo lo que contenga

rm -ri [carpeta] - borra todo, pero va preguntando qué documentos queremos borrar

rm -rf [carpeta] - borra todo sin consultar y lo elimina para siempre

Wildcards

* - hace referencia a 0 o más caracteres, no importa lo que haya antes o después, el comodín indica cualquiera

  • ls *.txt - muestra todos los archivos que sean .txt
  • ls prueba* - muestra todo lo que empiece por "prueba" sin importar lo siguiente

? - hace referencia a un número exacto de caracteres, teniendo en cuenta la extensión

  • ls ????* - muestra los archivos con los ? indicados o más (al tener el *)
  • ls ???? - muestra los archivos con los ? indicados

Listados avanzados

tree - muestra un árbol de directorios de nuestra ubicación actual

tree -a - muestra también los archivos ocultos

find [ruta] -name "nombre_archivo" - busca, en la ruta indicada, un archivo con el nombre indicado

Lectura de archivos

cat [archivo]- lee el archivo en la consola

less [archivo] - no carga todo el fichero, lo pagina en un lector para no tenerlo completo en la consola

more [archivo] - igual que el less, pero solo moviéndonos con las flechas y sin abrir un lector

head (-n número) [archivo] - muestra las 10 primeras líneas de un archivo por defecto, o las indicadas en el -n

tail (-n número) [archivo] - muestra las 10 últimas líneas de un archivo por defecto, o las indicadas en el -n

taul -f [archivo] - útil para logs, muestra el archivo IRL según se va modificando

Búsqueda y recuento

grep- búsqueda de texto en archivos

  • grep "texto" [archivo] - busca el texto en el archivo indicado, tal cual está escrito
  • grep -i "texto" [archivo] - hace lo mismo, pero ignorando mayúsculas y minúsculas
  • grep -r "texto" [ruta] - busca el texto indicado en todos los archivos de la carpeta, de manera recursiva

wc - busca información del archivo en bytes

  • wc -l [archivo] - muestra el número de líneas del archivo
  • wc -w [archivo] - muestra el número de palabras del archivo
  • wc -c [archivo] - muestra el número de caracteres del archivo

Redirecciones y pipes

> - redirecciona la salida de un comando hacia un archivo que indiquemos

  • echo "texto" > [ruta] - introduce la salida del primer comando en el archivo de la ruta que indiquemos, sobreescribiendo el archivo
  • echo "texto" >> [ruta] - escribe a continuación del archivo, sin borrar lo que hay

< - toma la entrada del archivo indicado

  • sort < [archivo] - ordena lo que hay dentro del archivo en consola

|| - concatena diferentes comandos

Variables de entorno

name="Daniel" - crea una variable de entorno con ese nombre, que luego podemos invocar con $name. Solo existe en esa sesión

export name="Daniel" - crea una variable de entorno y la exporta. Pero todavía no es utilizable

nano ~/.bash_profile - ponemos un "export variable="contenido" y guardamos. Ya se habrá guardado

Cómo unirse a un dominio en Ubuntu/Debian

  1. sudo apt update
  2. Instalar paquetes necesarios: sudo apt install -y [samba] [krb5-user] [winbind] [libnss-winbind] [libpam-winbind]
  3. Una vez finalizado, se abrirá un asistente:
    • Paso 1: poner el dominio (prueba.local, por ejemplo)
    • Paso 2: poner el nombre del servidor + el dominio (servidor.prueba.local, por ejemplo)
    • Paso 3: volvemos a poner el mismo que en el paso anterior
  4. sudo nano /etc/nsswitch.conf
  5. En ese archivo, en las líneas de "passwd" y "group" ponemos winbind al final de cada una, tras el "systemd"
  6. Configurar fichero samba. Primero, creamos una copia del archivo /etc/samba/smb.conf para evitar problemas. Una vez creada la copia, podemos borrar el original y crear uno nuevo que se llame smb.conf. Una vez abierto, copiamos lo siguiente:
    [global]
      workgroup = [nombre_grupo(dominio sin el .local)]
      server string = Samba Server Version %v
      security = ads
      realm = [dominio]
      socket options = TCP_NODELAY IPTOS_LOWDELAY SO_RCVBUF=131072 SO_SNDBUF=131072
      use sendfile = true

      idmap config * : backend = tdb
      idmap config * : range = 100000-299999
      idmap config TEST : backend = rid
      idmap config TEST : range = 10000-99999
      winbind separator = +
      winbind enum users = yes
      winbind use default domain = yes
      winbind refresh tickets = yes

      restrict anonymous = 2
      log file = /var/log/samba/log.%m
      max log size = 50
  7. Poner como DNS el servidor principal: sudo nano /etc/netplan/00-installer-config.yaml. Dentro del archivo, lo dejamos así:
    network:
      ethernets:
        [nombre_adaptador_red]:
          addresses:
          - [IP/máscara]
          gateway4: [gateway]
          nameservers:
            addresses:
            - [IP_servidor]
            search: [[dominio]]
      version: 2
  8. sudo netplan apply
  9. Comprobamos con un ping que los dispositivos se pueden conectar con el servidor Linux, tanto con IP como con el dominio, resolviendo por DNS.
  10. sudo kinit Administrador. Si no nos devuelve información, significa que está correcto.
  11. sudo klist. Esto nos devolverá el ticket, indicando que estamos validados por el kerberos.
  12. Unir definitivamente el Ubuntu Server al dominio: sudo net ads join -U Administrador@laboratorio.local
  13. Vamos al Windows Server -> DNS -> Zona de búsqueda directa -> doble click sobre el dominio -> botón derecho -> Agregar nuevo host:
    • Ponemos el nombre del servidor Ubuntu
    • Ponemos también la IP del servidor Ubuntu
  14. sudo /etc/init.d/winbind restart
  15. sudo wbinfo -u. Con esto podremos ver los usuarios creados en el dominio. Con -g veremos los grupos

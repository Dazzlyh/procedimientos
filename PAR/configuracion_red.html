Cómo configurar una red en Packet Tracer

Praparación previa

Identificar las redes. Si tenemos más de un router, miramos cuántos routers tenemos, ya que la red entre routers es una nueva. Tenemos el siguiente ejemplo:

La red entre routers solamente va a tener 2 dispositivos (en este caso), por tanto, es un /30, ya que solamente necesitamos 2 bits para 2 redes + IP de Red + Ip Broadcast (4 en total). En las otras subredes, dejaríamos /24 ya que es más probable que existan más dispositivos.

Configuración de la red

  1. Establecer la IP y máscara de red a cada dispositivo cliente.
  2. Establecemos también la IP y la máscara al router (o routers en caso de que haya varios). En el ejemplo de arriba, sería:
    • R1: 192.168.1.1
    • R2: 192.168.2.1
  3. Ahora toca configurar la IP para cada interfaz. Por ejemplo, en R1:
    • enable
    • configure terminal
    • interface GigabitEthernet 0/1 (la interfaz de la red de la izquierda)
    • ip address 192.168.1.1 255.255.255.0
    • no shutdown
    • exit
    • Esto se repite para cada interfaz de cada router. Para la interfaz 0/0 (la red entre ambos routers) sería 192.168.0.1 255.255.255.252 (al ser /30)
  4. Una vez establecidas las Gateways, tenemos que configurarla en cada uno de los dispositivos con su correspondiente.
  5. Ya tendríamos configurado cada router, pero faltan los saltos. En el ejemplo de arriba: queremos enviar un paquete desde el Server hasta PC4 que está en otra red. Para ello, el server envía un paquete a nuestro router y sabe a la red que tiene que llegar, pero no como. Por tanto, en los comandos tenemos que indicarle la IP de la red destino, la máscara de red (de la red destino) y la puerta de enlace (IP del otro router, de R2 en este caso):
    • enable
    • configure terminal
    • ip route 192.168.2.0 255.255.255.0 192.168.0.2
    • Esto se repite con todos los routers para cada red
  6. Si está todo correctamente, ya podríamos probar a realizar un ping a un dispositivo de otra red, y debería de funcionar.
